Volume and Projection Geometries¶

Tomosipo has six types of geometries with varying degrees of flexibility.

Geometries¶

Creation function

Geometry

tomosipo.parallel()

3D circular parallel beam geometry

tomosipo.parallel_vec()

3D arbitrarily-oriented parallel beam geometry

tomosipo.cone()

3D circular cone beam geometry

tomosipo.cone_vec()

3D arbitrarily-oriented cone beam geometry

tomosipo.volume()

3D axis-aligned volume geometry

tomosipo.cone_vec()

3D arbitrarily-oriented volume geometry. This object cannot be converted to ASTRA directly, but can be used in geometric computations.

Geometry creation¶

Circular projection geometries¶

The following conventions are used:

  1. When size is not provided, it is taken to be equal to the shape, i.e., the detector pixel size is equal to one in each dimension by default.

  2. The angles be provided as a single integer. This is automatically expanded to a half circle arc (ts.parallel()) or full circle arc (ts.cone()).

  3. An array of angles can also be provided, in units of radians.

  4. The size and shape parameters can be provided as a single float, resulting in a square detector, or as a tuple containing the height and width of the detector.

tomosipo.parallel(*, angles=1, shape=1, size=None)[source]¶

Create a parallel-beam geometry

Parameters

  • angles –

    np.array or integral value

    If integral value: the number of angles in the parallel-beam geometry. This describes a half arc (pi radians) with uniform placement and without the start and end point overlapping.

    If np.array: the values of the array are taken as projection angle (units are radians).

  • shape – (int, int) or int The detector shape in pixels. If tuple, the order is (height, width). Else the pixel has the same number of pixels in the U and V direction.

  • size –

    (float, float) or float The detector size. If a single float is provided, the detector is square with equal width and height.

    The order is (height, width), i.e. (v, u).

Returns

A circular parallel-beam geometry

Return type

ParallelGeometry

tomosipo.cone(*, angles=1, shape=1, 1, size=None, cone_angle=None, src_orig_dist=None, src_det_dist=None)[source]¶

Create a cone-beam geometry

Parameters

  • angles –

    np.array or integral value If integral value: the number of angles in the cone-beam geometry. This describes a full arc (2 pi radians) with uniform placement and without the start and end point overlapping.

    If np.array: the values of the array are taken as projection angle (units are radians).

  • shape – (int, int) or int The detector shape in pixels. If tuple, the order is (height, width). Else the pixel has the same number of pixels in the U and V direction.

  • size –

    (float, float) or float The detector size. If a single float is provided, the detector is square with equal width and height.

    The order is (height, width), i.e. (v, u).

  • cone_angle – scalar The fraction det_height / src_det_dist. If provided, src_orig_dist and src_det_dist need not be provided. The detector is placed on the origin.

  • src_orig_dist – scalar The source to origin distance.

  • src_det_dist – The source to detector distance.

Returns

a cone-beam geometry

Return type

ConeGeometry

Arbitrary projection geometries¶

tomosipo.parallel_vec(*, shape, ray_dir, det_pos, det_v, det_u)[source]¶

Create a parallel-beam vector geometry

Parameters

  • shape – (int, int) or int The detector shape in pixels. If tuple, the order is (height, width). Else the pixel has the same number of pixels in the U and V direction.

  • ray_dir – np.array A numpy array of dimension (num_positions, 3) with the ray direction in (Z, Y, X) order.

  • det_pos – A numpy array of dimension (num_positions, 3) with the detector center positions in (Z, Y, X) order.

  • det_v – A numpy array of dimension (num_positions, 3) with the vector pointing from the detector (0, 0) to (1, 0) pixel (up).

  • det_u – A numpy array of dimension (num_positions, 3) with the vector pointing from the detector (0, 0) to (0, 1) pixel (sideways).

Returns

An arbitrarily oriented parallel-beam geometry

Return type

ParallelVectorGeometry

Volume geometries¶

tomosipo.volume(*, shape=1, 1, 1, pos=None, size=None, extent=None)[source]¶

Create an axis-aligned volume geometry

A VolumeGeometry is an axis-aligned cuboid centered on pos.

You may provide a combination of arguments to create a new volume geometry: - shape (size will equal shape) - shape and pos (size will equal shape) - shape and size (volume will be centered on the origin) - shape, pos, and size. - shape and extent

Parameters

  • shape – int or (int, int, int) Shape of the voxel grid underlying the volume.

  • extent – (scalar, scalar) or ((scalar, scalar), (scalar, scalar), (scalar, scalar)) The minimal and maximal value of the volume in the Z, Y, X coordinate. If only one minimal and maximal value is provided, then it is applied to all coordinates.

  • center – scalar or (scalar, scalar, scalar)

  • size – scalar or (scalar, scalar, scalar)

Returns

a volume geometry of the unit cube with shape as given.

Return type

VolumeGeometry

tomosipo.volume_vec(*, shape, pos=0, w=1, 0, 0, v=0, 1, 0, u=0, 0, 1)[source]¶

Create a new volume vector geometry

Like the parallel and cone vector geometries, the volume vector geometry can be arbitrarily oriented and positioned.

The position describes the center of the volume.

Parameters

  • shape – (int, int, int) or int The shape of the volume as measured in basis elements w, v, u.

  • pos – scalar, np.array A numpy array of dimension (num_steps, 3) describing the center of the volume in world-coordinates (Z, Y, X). You may also pass a 3-tuple or a scalar.

  • w – np.array (optional) A numpy array of dimension (num_steps, 3) describing the w basis element in (Z, Y, X) coordinates. Default is (1, 0, 0).

  • v – np.array (optional) A numpy array of dimension (num_steps, 3) describing the v basis element in (Z, Y, X) coordinates. Default is (0, 1, 0).

  • u – np.array (optional) A numpy array of dimension (num_steps, 3) describing the u basis element in (Z, Y, X) coordinates. Default is (0, 0, 1).

Returns

Return type
